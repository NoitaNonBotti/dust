GET /api/health/

Use this as a lightweight readiness check before starting frontend API work.

PowerShell Example

curl.exe http://localhost:8000/api/health/

Parameters

None.

Example Response

{
  "ok": true,
  "service": "dust_django"
}

Optional Error Handling

  • Connection refused: Django is not running or is not on port 8000.
GET /api/lost-items/

Fetches the current item collection for the Lost Items and Admin screens.

PowerShell Example

curl.exe http://localhost:8000/api/lost-items/

Parameters

None.

Example Response

{
  "items": [
    {
      "id": "1",
      "name": "Blue Backpack",
      "category": "Bags",
      "description": "Navy blue backpack found near the cafeteria.",
      "location": "Cafeteria",
      "dateFound": "2026-05-07",
      "status": "unclaimed",
      "imageUrl": "/media/lost_items/example.png"
    }
  ]
}

Optional Error Handling

  • Connection refused: Django is not running or is not on port 8000.
POST /api/lost-items/

Creates an item record from the reporting workflow, including optional local image storage.

PowerShell JSON Example

curl.exe -X POST http://localhost:8000/api/lost-items/ `
  -H "Content-Type: application/json" `
  -d "{\"name\":\"Blue Backpack\",\"category\":\"Bags\",\"description\":\"Navy blue backpack found near the cafeteria.\",\"location\":\"Cafeteria\",\"imageUrl\":\"\"}"

PowerShell Image Upload Example

curl.exe -X POST http://localhost:8000/api/lost-items/ `
  -F "name=Blue Backpack" `
  -F "category=Bags" `
  -F "description=Navy blue backpack found near the cafeteria." `
  -F "location=Cafeteria" `
  -F "image=@C:\path\to\image.png"

Parameters

Name Required Description
name Yes Short item name.
category Yes Single category stored by the basic backend model.
description Yes Details that help identify the item.
location Yes Where the item was found or can be retrieved.
dateFound No Date in YYYY-MM-DD format. Defaults to today's server date.
imageUrl No Existing image URL/path for JSON requests.
image No Multipart image file. Supported types: JPEG, PNG, GIF, WebP.

Optional Error Handling

  • 400 Invalid JSON body.: request body was not valid JSON.
  • 400 Missing required fields.: one or more required fields were empty.
  • 400 Image must be a JPEG, PNG, GIF, or WebP file.: unsupported uploaded image type.
PATCH /api/lost-items/<id>/

Applies admin dashboard edits to one item, including status and optional image replacement.

PowerShell Example

curl.exe -X PATCH http://localhost:8000/api/lost-items/1/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-ADMIN-PASSWORD: password" `
  -d "{\"name\":\"Updated Backpack\",\"category\":\"Bags\",\"description\":\"Updated description\",\"location\":\"Library\",\"dateFound\":\"2026-05-07\",\"status\":\"claimed\"}"

Parameters

Name Required Description
id Yes Lost item ID in the URL path.
X-DUST-ADMIN-PASSWORD Yes Temporary admin header. Current development value is password.
name, category, description, location No Text fields to update.
dateFound No Date in YYYY-MM-DD format.
status No One of unclaimed, claimed, or returned.
imageUrl or image No Replacement image path/URL or multipart image file.

Optional Error Handling

  • 401 Admin password is required.: missing or incorrect temporary admin header.
  • 404 Not Found: no lost item exists with that ID.
  • 400 Invalid JSON body.: request body was not valid JSON.
  • 400 Invalid item status.: status was not one of the accepted values.
  • 400 Image must be a JPEG, PNG, GIF, or WebP file.: unsupported uploaded image type.
DELETE /api/lost-items/<id>/

Removes an item from the database when an admin decides the record should no longer appear.

PowerShell Example

curl.exe -X DELETE http://localhost:8000/api/lost-items/1/ `
  -H "X-DUST-ADMIN-PASSWORD: password"

Parameters

Name Required Description
id Yes Lost item ID in the URL path.
X-DUST-ADMIN-PASSWORD Yes Temporary admin header. Current development value is password.

Example Response

{
  "deleted": true
}

Optional Error Handling

  • 401 Admin password is required.: missing or incorrect temporary admin header.
  • 404 Not Found: no lost item exists with that ID.
