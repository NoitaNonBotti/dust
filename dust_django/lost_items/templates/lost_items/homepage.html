DUST API Reference

Development reference for the Django REST-style endpoints used by the React frontend.

Health JSON Items JSON Django Admin React Frontend

Base URL

http://localhost:8000

Authentication

Most write endpoints use a session token returned by login. Send it on every authenticated request: 

X-DUST-SESSION: <token from login response>

Roles

Role How to obtain Typical access
guest No header (browse only) List items, file low-priority claims, guest inquiries
student POST /api/auth/gbox/ with Google ID token Report items, file high-priority claims, cancel own claims
admin POST /api/auth/admin/ or create_dust_admin Edit/delete items, approve claims, manage templates

GBox login requires GOOGLE_CLIENT_ID on the Django server. Optional GBOX_ALLOWED_DOMAIN restricts student emails (for example @school.edu).

GET /api/health/

Simple readiness check for the Django API.

PowerShell Example

curl.exe http://localhost:8000/api/health/

Example Response

{
  "ok": true,
  "service": "dust_django",
  "database": "ok"
}

Errors

  • 503 — database unreachable (ok: false, database: "error").
POST /api/auth/gbox/

Verifies a Google ID token and creates a student session.

PowerShell Example

curl.exe -X POST http://localhost:8000/api/auth/gbox/ `
  -H "Content-Type: application/json" `
  -d "{\"credential\":\"GOOGLE_ID_TOKEN_FROM_FRONTEND\"}"

JSON Body

NameRequiredDescription
credential Yes Google Sign-In ID token from the React client.

Example Response

{
  "user": {
    "token": "a1b2c3...",
    "role": "student",
    "name": "Jane Student",
    "email": "jane@school.edu"
  }
}

Errors

  • 400 — missing credential or invalid JSON.
  • 401 — token verification failed or email not allowed.
POST /api/auth/admin/

Authenticates a console-managed admin account and returns a session token.

PowerShell Example

curl.exe -X POST http://localhost:8000/api/auth/admin/ `
  -H "Content-Type: application/json" `
  -d "{\"username\":\"admin\",\"password\":\"your-password\"}"

Create admins with: python manage.py create_dust_admin

Example Response

{
  "user": {
    "token": "d4e5f6...",
    "role": "admin",
    "name": "admin",
    "email": ""
  }
}

Errors

  • 401 — invalid username or password.
POST /api/auth/logout/

Deletes the session matching X-DUST-SESSION.

PowerShell Example

curl.exe -X POST http://localhost:8000/api/auth/logout/ `
  -H "X-DUST-SESSION: YOUR_TOKEN_HERE"

Example Response

{ "ok": true }

POST /api/lost-items/

Creates an item from the Report Item workflow. Requires a student or admin session.

PowerShell JSON Example

curl.exe -X POST http://localhost:8000/api/lost-items/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-SESSION: YOUR_TOKEN_HERE" `
  -d "{\"name\":\"Blue Backpack\",\"category\":\"Bags\",\"description\":\"Navy blue backpack.\",\"location\":\"Cafeteria\",\"imageUrl\":\"\"}"

PowerShell Image Upload Example

curl.exe -X POST http://localhost:8000/api/lost-items/ `
  -H "X-DUST-SESSION: YOUR_TOKEN_HERE" `
  -F "name=Blue Backpack" `
  -F "category=Bags" `
  -F "description=Navy blue backpack found near the cafeteria." `
  -F "location=Cafeteria" `
  -F "image=@C:\path\to\image.png"

Parameters

NameRequiredDescription
X-DUST-SESSIONYesStudent or admin session token.
nameYesShort item name.
categoryYesSingle category string.
descriptionYesItem details.
locationYesWhere the item was found.
dateFoundNoYYYY-MM-DD. Defaults to today.
imageUrlNoImage path/URL for JSON requests.
imageNoMultipart file (JPEG, PNG, GIF, WebP).

Errors

  • 401 — student or admin login required.
  • 400 — invalid JSON, missing fields, or bad image type.
PATCH /api/lost-items/<id>/

Admin dashboard edits for one item, including status and optional image replacement.

PowerShell Example

curl.exe -X PATCH http://localhost:8000/api/lost-items/1/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-SESSION: ADMIN_TOKEN_HERE" `
  -d "{\"name\":\"Updated Backpack\",\"category\":\"Bags\",\"description\":\"Updated description\",\"location\":\"Library\",\"dateFound\":\"2026-05-07\",\"status\":\"claimed\"}"

Parameters

NameRequiredDescription
idYesLost item ID in the URL path.
X-DUST-SESSIONYesAdmin session token.
name, category, description, locationNoText fields to update.
dateFoundNoYYYY-MM-DD.
statusNounclaimed, claimed, or returned.
imageUrl or imageNoReplacement image.

Errors

  • 401 — admin login required.
  • 404 — item not found.

Example Response

POST /api/claims/

Files a claim on an unclaimed item only. With a student session the claim gets priority: "high". With no session (guest) it gets priority: "low". Once an item is claimed or returned, new claims return 409.

Student claim

curl.exe -X POST http://localhost:8000/api/claims/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-SESSION: STUDENT_TOKEN_HERE" `
  -d "{\"itemId\":\"1\",\"claimantName\":\"Jane Student\",\"claimantEmail\":\"jane@school.edu\",\"claimantPhone\":\"555-0100\",\"description\":\"It has my ID inside.\"}"

Guest claim (no session)

curl.exe -X POST http://localhost:8000/api/claims/ `
  -H "Content-Type: application/json" `
  -d "{\"itemId\":\"1\",\"claimantName\":\"Guest User\",\"claimantEmail\":\"guest@example.com\",\"claimantPhone\":\"\",\"description\":\"I think this is mine.\"}"

JSON Body

NameRequiredDescription
itemIdYesLost item ID.
claimantNameYesClaimant full name.
claimantEmailYesContact email.
claimantPhoneNoContact phone (recommended for students).
descriptionYesWhy the item belongs to the claimant.

Example Response

{
  "id": "5",
  "itemId": "1",
  "claimantName": "Guest User",
  "claimantEmail": "guest@example.com",
  "claimantPhone": "",
  "description": "I think this is mine.",
  "dateSubmitted": "2026-05-07",
  "status": "pending",
  "priority": "low",
  "createdByToken": ""
}
PATCH /api/claims/<id>/

Updates claim status. Approving a claim sets the parent item status to claimed.

Admin approve

curl.exe -X PATCH http://localhost:8000/api/claims/5/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-SESSION: ADMIN_TOKEN_HERE" `
  -d "{\"status\":\"approved\"}"

Student cancel own claim

curl.exe -X PATCH http://localhost:8000/api/claims/5/ `
  -H "Content-Type: application/json" `
  -H "X-DUST-SESSION: STUDENT_TOKEN_HERE" `
  -d "{\"status\":\"cancelled\"}"

Status values

pending, approved, rejected, cancelled

Permissions

  • approved / rejected — admin only.
  • cancelled — claim owner (matching session token) or admin.
  • Guest claims have an empty createdByToken; only admins can cancel them.
POST /api/guest-inquiries/

Legacy guest assistance endpoint. Still accepted for compatibility; creates a low-priority claim and a GuestInquiry audit row. Prefer POST /api/claims/ without a session for new clients.

PowerShell Example

curl.exe -X POST http://localhost:8000/api/guest-inquiries/ `
  -H "Content-Type: application/json" `
  -d "{\"itemId\":\"1\",\"contactName\":\"Guest User\",\"contactEmail\":\"guest@example.com\",\"contactPhone\":\"\",\"message\":\"Please help me verify ownership.\"}"

JSON Body

NameRequiredDescription
itemIdYesLost item ID.
contactNameYesGuest name.
contactEmailYesGuest email.
contactPhoneNoGuest phone.
messageYesAssistance request message.

Returns the same claim object shape as POST /api/claims/.
